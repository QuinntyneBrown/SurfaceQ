# L1 - High-Level Requirements

## L1-001: Explicit public-api.ts generation
SurfaceQ must generate a fully explicit, deterministic `public-api.ts` file for an Angular library, replacing wildcard re-exports with named re-exports of the form `export { Foo, Bar } from './path'`.

## L1-002: CLI surface with three subcommands
SurfaceQ must expose `generate`, `check`, and `diff` subcommands implemented via `System.CommandLine`, each supporting an optional `--project <path>` argument and using documented exit codes.

## L1-003: Project and source file discovery
SurfaceQ must discover the target Angular library by locating `ng-package.json`, resolving its `entryFile`, and recursively discovering `.ts` files under the resolved scan root while applying documented exclusions.

## L1-004: Symbol discovery via TypeScript Compiler API
SurfaceQ must delegate TypeScript parsing to a bundled Node.js sidecar running the official TypeScript Compiler API, communicating over newline-delimited JSON on stdio.

## L1-005: Deterministic rendering and output format
SurfaceQ must render the output file in a deterministic format that is byte-identical across runs, hosts, and operating systems for identical inputs.

## L1-006: CI-friendly drift detection
SurfaceQ must support CI pipelines with a single non-zero exit code on drift, no network activity, and no interactive prompts.

## L1-007: .NET global tool distribution
SurfaceQ must be distributed as a .NET global tool via NuGet, built on .NET 8.0, and runnable on Windows, Linux, and macOS (x64 and ARM64) with a bundled Node.js runtime requiring no external user-installed dependencies.

## L1-008: Performance and resource bounds
SurfaceQ must meet documented performance and memory targets for libraries up to 500 files.

## L1-009: Logging and observability
SurfaceQ must emit plain-text logs via `Microsoft.Extensions.Logging` with configurable verbosity and correct stream routing (errors/warnings to stderr, info to stdout).

## L1-010: Security and isolation
SurfaceQ must not perform network I/O, telemetry, or shell invocation, and must pass only file paths (never file content) as sidecar arguments.

## L1-011: Error handling and edge cases
SurfaceQ must produce clear, actionable errors and warnings for documented edge cases including missing/malformed manifests, default exports, syntax errors, permission errors, and empty scan roots.

## L1-012: Configuration minimalism
SurfaceQ v1 must operate solely from `ng-package.json` auto-detection plus CLI flags, with no project-level config file.
